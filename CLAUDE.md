# Instructions

## On Startup
- Always read docs/PROJECT_NOTES.md first to understand the project context and current state.
- Always keep project notes updated with any changes or decisions made.
- Always keep docs/CHANGELOG.md up to date with any changes or decisions made.
- Always keep OpenAPI schema (../ospchat-shared/docs/) up to date with any API changes.
- Always use the feature-dev skill when developing new features.
- Do not bump up VERSION unless specifically instructed.
- Do not use co-authored ever.

## Project Commands
- Lint: `make lint` - non-zero exit code is a failure.
- Build: `make build`

## Important Files
- docs/PROJECT_NOTES.md - Contains project overview, goals, and current implementation status
- Makefile - Contains build, test, and lint commands
- README.md - Contains basic project information.
